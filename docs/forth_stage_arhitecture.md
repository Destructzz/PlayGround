### Архитектура системы

Проектируемая автоматизированная информационная система досугового центра «PlayGround» реализуется на основе клиент-серверной архитектуры с использованием централизованной базы данных.

Система включает в себя следующие основные компоненты:

- **Клиентская часть** — пользовательский интерфейс, доступный через веб-приложение. Клиентская часть предназначена для взаимодействия пользователей с системой и обеспечивает доступ к функционалу в зависимости от роли пользователя.
- **Серверная часть** — сервер приложений, реализующий бизнес-логику системы, обработку запросов клиентов, управление доступом и взаимодействие с базой данных.
- **База данных** — централизованное хранилище информации, используемое для хранения данных о пользователях, помещениях, услугах, тарифах, бронированиях и оплатах.

В системе предусмотрены различные роли пользователей, каждая из которых имеет собственный набор функций:

- **Клиент** — пользователь системы, имеющий возможность самостоятельной регистрации через пользовательский интерфейс, просмотра доступных услуг, помещений и создания бронирований.
- **Продавец (сотрудник)** — пользователь системы, осуществляющий регистрацию клиентов, оформление бронирований и ввод данных в систему в ручном режиме при обслуживании посетителей на месте.
- **Администратор** — пользователь системы, обладающий расширенными правами доступа, включая управление конфигурацией здания, настройку игровых зон и помещений, изменение тарифов и цен, а также администрирование пользователей системы.

Клиентская часть взаимодействует с серверной частью посредством запросов к программному интерфейсу приложения. Серверная часть осуществляет обработку запросов, проверку прав доступа, выполнение бизнес-логики и обращение к базе данных. Такое архитектурное решение обеспечивает целостность данных, разграничение доступа и удобство сопровождения системы.
