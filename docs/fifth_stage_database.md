### 5. Проектирование базы данных

На этапе проектирования базы данных разрабатывается информационная модель автоматизированной информационной системы досугового центра «PlayGround». Основной целью данного этапа является определение структуры хранения данных, обеспечивающей целостность, непротиворечивость и удобство обработки информации.

В процессе проектирования определяются основные сущности системы, их атрибуты и связи между ними. База данных предназначена для хранения информации о пользователях системы, сотрудниках, помещениях, услугах, бронированиях и оплатах.

Информационная модель строится с учётом принципов нормализации данных, что позволяет исключить избыточность информации и обеспечить корректность операций добавления, изменения и удаления данных.

Основными сущностями базы данных являются:
- users — пользователи;
- роли — роли в системе;
- staff — сотрудники;
- игровые зоны и помещения;
- тип зоны;
- предоставляемые услуги;
- бронирования ресурсов;
- данные об оплатах;
- журнал действий пользователей.


#### Пользователь системы (users)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | uuid | Уникальный идентификатор пользователя |
| google_id | varchar | Уникальный идентификатор пользователя в Google OAuth |
| full_name | text | Имя и фамилия пользователя (получается из Google, может быть изменено) |
| email | varchar | Адрес электронной почты пользователя (получается из Google, может быть изменён) |
| avatar_url | varchar | Ссылка на аватар пользователя (получается из Google) |
| phone | varchar | Контактный номер телефона пользователя |
| role | role | Роль пользователя в системе (администратор, продавец, клиент) |
| is_active | boolean | Статус активности пользователя |
| created_at | datetime | Дата регистрации пользователя |
| updated_at | datetime | Дата последнего обновления данных |
| deleted_at | datetime | Дата удаления пользователя |


#### Роли (role)

Для определения ролей пользователей в системе используется перечисляемый тип (ENUM) базы данных PostgreSQL. Данный подход позволяет явно ограничить набор допустимых значений и повысить целостность данных.

Перечень ролей пользователей:
- `admin` — администратор системы;
- `seller` — сотрудник (продавец), осуществляющий регистрацию клиентов и оформление услуг;
- `client` — клиент досугового центра.

#### Игровые зоны и помещения (zones)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор зоны |
| name | varchar | Название игровой зоны или помещения |
| zone_type | zone_type | Тип зоны |
| capacity | integer | Максимальная вместимость зоны |
| description | text | Описание зоны |
| is_active | boolean | Статус доступности зоны |
| created_at | datetime | Дата создания записи |
| updated_at | datetime | Дата последнего изменения |

#### Тип зоны (zone_type)

Перечень типов зон:
- `game` — игровая зона;
- `event` — зона для проведения мероприятий;
- `vip` — зона повышенного комфорта (VIP);
- `lounge` — зона отдыха;
- `sys` — системные зоны (служебные и вспомогательные помещения).


#### Предоставляемые услуги (services)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор услуги |
| name | varchar | Название услуги |
| zone_id | integer | Связанная игровая зона или помещение |
| duration | integer | Длительность услуги (в минутах) |
| price | decimal | Стоимость услуги |
| currency | varchar | Валюта цены (по умолчанию RUB) |
| description | text | Описание услуги |
| is_active | boolean | Статус доступности услуги |
| created_at | datetime | Дата создания записи |
| updated_at | datetime | Дата последнего изменения |

#### Бронирования ресурсов (bookings)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор бронирования |
| user_id | uuid | Пользователь (клиент), создавший бронирование |
| zone_id | integer | Забронированная игровая зона или помещение |
| service_id | integer | Выбранная услуга |
| start_time | datetime | Дата и время начала бронирования |
| end_time | datetime | Дата и время окончания бронирования |
| participants | integer | Количество участников |
| total_price | decimal | Итоговая стоимость бронирования |
| status | booking_status | Статус бронирования (создано, подтверждено, отменено) |
| created_at | datetime | Дата создания бронирования |
| updated_at | datetime | Дата последнего изменения |

#### Статусы бронирования (booking_status)

- `created` — создано;
- `confirmed` — подтверждено;
- `canceled` — отменено;
- `completed` — оказано или завершено.

#### Сотрудники (staff)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор сотрудника |
| user_id | uuid | Учётная запись пользователя системы |
| position | position_type | Должность (администратор, продавец, оператор и т.п.) |
| hire_date | date | Дата приёма на работу |
| phone | varchar | Контактный номер |
| email | varchar | Рабочий адрес электронной почты |
| is_active | boolean | Статус активности сотрудника |
| created_at | datetime | Дата создания записи |
| updated_at | datetime | Дата последнего обновления |
| deleted_at | datetime | Дата удаления записи |

#### Должности сотрудников (position_type)

- `admin` — управляющий площадкой;
- `seller` — сотрудник фронт-офиса/кассир;
- `operator` — оператор бронирований/поддержки;
- `tech` — технический специалист.

#### Оплаты (payments)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор платежа |
| booking_id | integer | Бронирование, за которое проводится оплата |
| amount | decimal | Сумма платежа |
| currency | varchar | Валюта платежа |
| payment_method | payment_method | Способ оплаты (наличные, карта, онлайн) |
| status | payment_status | Статус платежа |
| receipt_number | varchar | Номер чека/квитанции или слип операции |
| paid_at | datetime | Дата и время проведения оплаты |
| created_at | datetime | Дата создания записи |
| updated_at | datetime | Дата последнего обновления |

#### Способы оплаты (payment_method)

- `cash` — наличные;
- `card` — банковская карта;
- `online` — онлайн-оплата.

#### Статусы оплаты (payment_status)

- `pending` — ожидает проведения;
- `paid` — успешно оплачено;
- `failed` — отклонено или ошибка;
- `refunded` — произведён возврат средств.

#### Журнал действий пользователей (audit_logs)

| Поле | Тип данных | Описание |
|------|-----------|----------|
| id | integer | Уникальный идентификатор записи |
| user_id | uuid | Пользователь, совершивший действие |
| action | action_type | Тип действия (создание, обновление, удаление, вход) |
| entity | entity_type | Сущность, к которой относится действие (booking, service и т.п.) |
| entity_id | integer | Идентификатор сущности |
| payload | jsonb | Детали изменения или параметры запроса |
| created_at | datetime | Дата и время фиксации действия |
| ip_address | varchar | IP-адрес клиента |

#### Тип действия (action_type)

- `create` — создание записи;
- `update` — обновление записи;
- `delete` — удаление записи;
- `login` — вход пользователя;
- `logout` — выход пользователя.

#### Тип сущности (entity_type)

- `booking` — бронирование;
- `service` — услуга;
- `zone` — игровая зона или помещение;
- `payment` — оплата;
- `user` — пользователь системы;
- `staff` — сотрудник;
- `other` — прочие действия.
