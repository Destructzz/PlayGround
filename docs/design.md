# Дизайн-референс (как на скрине) — правила стиля, чтобы применить к ЛЮБОМУ контенту

Цель: не копировать текст/пункты, а **перенести визуальную систему** (цвета, формы, сетки, эффекты, состояния) на ваш контент AИС «PlayGround».

---

## 1) Общая визуальная система

### Стиль
- **Тёмный premium SaaS**: строгий, “дорогой”, технологичный.
- Визуальная метафора: **glassmorphism** (полупрозрачные карточки + blur) поверх тёмного фона.
- Много воздуха: крупные отступы, спокойная иерархия, без лишнего декора.

### Типографика
- Шрифт: современный sans-serif (Inter/Manrope).
- Иерархия:
  - Навигация: 14–16px, medium.
  - Заголовки в меню/карточках: 14–15px semibold.
  - Подписи/описания: 12–13px regular, сниженный контраст.
  - Заголовки секций: 10–12px uppercase, letter-spacing повышен.

---

## 2) Цветовая палитра (переносим 1:1)

### Базовые слои
- **Фон страницы**: почти чёрный (угольный), без чистого #000.  
  Пример ощущения: `#0B0C0F` → `#0D0D10`.
- **Топ-бар**: такой же тёмный, чуть более плотный/ровный.
- **Поверхности (панели/карточки)**: тёмно-серые с прозрачностью (не “плашка”, а “стекло”).

### Текст и контраст
- Основной текст: почти белый (90–95%).
- Вторичный текст: серый (50–60%).
- Заголовки секций: серый (40–50%), uppercase.

### Акценты
- Акцент НЕ кричащий (без ярких цветов).  
  Акцент достигается:
  - разницей контрастов,
  - свечением/hover,
  - прозрачными бейджами,
  - иконками.

---

## 3) Фон и глубина (важно для “как на референсе”)

### Background layer
- На фоне должны быть:
  - мягкие градиенты/затемнения,
  - **абстрактный большой объект справа** (рендер/форма/фото с размытием).
- Этот объект НЕ должен спорить с UI:
  - затемнён (overlay 60–75%),
  - слегка размыт,
  - воспринимается как “декорация”.

### Принцип глубины
- Фон → затемнение → стеклянная панель (mega menu) → контент внутри.
- Тень у панели широкая и мягкая, чтобы UI “висел” над фоном.

---

## 4) Navbar (верхняя панель) — форма и поведение

### Компоновка
- Слева: 3–5 пунктов меню.
- Справа: 1 action-ссылка/кнопка (“Contact Us” по стилю).

### Вид пунктов
- Текст пунктов светло-серый, при hover — белее.
- Пункты с выпадашками имеют маленький chevron вниз.
- Активный пункт (открыт dropdown):
  - чуть ярче,
  - возможно лёгкая подсветка (не подчёркивание ярким цветом, а subtle).

---

## 5) Mega-dropdown (ключевой паттерн)

### Внешний вид панели
- Большая прямоугольная панель со скруглением **16–20px**.
- Фон: тёмный, полупрозрачный + **backdrop blur**.
- Тонкая граница 1px с низкой непрозрачностью (около 6% белого).
- Тень: широкая, мягкая, без резких краёв.

### Сетка внутри
- Десктоп: **3 колонки**.
- Межколоночный gap заметный (примерно 24–32px).
- Внутри колонок — блоки/секции с заголовком и списком элементов.

### Заголовки секций
- Маленький uppercase, серый, letter-spacing увеличен.
- Всегда перед списком, отделён от него небольшим отступом.

---

## 6) Элемент списка (универсальный “строковый пункт”)

Этот элемент используйте для ваших сущностей (Зоны, Бронирования, Тарифы, Отчёты, Пользователи и т.д.).

### Структура (строго)
- Слева: квадратный контейнер с иконкой.
- Справа: вертикальный текстовый блок:
  - строка 1: название (semibold)
  - строка 2: короткое описание/подпись (серое)
- Иногда: бейдж “NEW” или “BETA” рядом с названием (прозрачная пилюля).

### Геометрия
- Высота строки: 48–56px.
- Иконка-контейнер: 32–36px, скругление 10–12px.
- Gap иконка↔текст: 12–14px.
- Текст не должен “прыгать”: ограничить описание до 1 строки (clamp).

### Hover
- При наведении строка получает:
  - лёгкий фон (очень слабый, 3–5% белого),
  - чуть ярче title,
  - контейнер иконки чуть светлее.
- Никаких ярких заливок — всё subtle.

---

## 7) Бейджи (NEW / статус)

Использовать как системный элемент для вашего контента:
- “NEW”, “BETA”, “Soon”, “Admin”, “Client”, “VIP”.

### Стиль бейджа
- Маленькая pill-форма:
  - высота 16–18px,
  - скругление 999px,
  - фон: белый 8–12% прозрачности,
  - текст: серый 70–80%,
  - uppercase 10–11px.

---

## 8) Компоненты, которые должны выглядеть “как в референсе”

### Карточки/панели
- Всегда:
  - тёмный фон,
  - тонкая border,
  - мягкая тень,
  - скругления 16px+,
  - иногда blur.

### Поля ввода/формы
- Инпуты тёмные, border слабый, focus — мягкий (не кислотный).
- Плейсхолдер серый, текст белее.
- Ошибки/валидность — без ярких красных заливок, лучше тонкая подсветка.

### Кнопки
- Основная: тёмная/нейтральная с лёгкой подсветкой на hover.
- Вторичная: outline/ghost.
- Никаких ярких цветных кнопок по умолчанию.

### Иконки
- Линейные (outline), тонкий stroke.
- Иконки должны быть одинакового стиля и размера.
- Использовать как “маркер” пункта/раздела.

---

## 9) Применение к контенту PlayGround (как переносить стиль)

### Как переводить сущности в UI элементы
- **Зоны/помещения** → item rows в mega-menu и карточках (“VR”, “Event Room”, “Lounge”).
- **Услуги** → item rows с коротким описанием (длительность/формат).
- **Тарифы** → карточки с названием и мелким описанием.
- **Бронирования** → списки/табличные строки в том же стиле (иконка + заголовок + метаданные).
- **Пользователи/роли** → бейджи роли рядом с именем.

### Основное правило
- Любой ваш контент должен “садиться” на:
  - стеклянные панели,
  - 3-колоночные списки,
  - строки с иконкой,
  - капс-заголовки секций,
  - мягкие hover-состояния.

---

## 10) Motion (микро-анимации)

- Dropdown появляется:
  - fade-in + лёгкий сдвиг вниз 6–10px,
  - 150–220ms, ease-out.
- Hover: без “скачков”, только изменение прозрачности/фона.

---

## 11) Быстрый чеклист “похоже на референс?”

- [ ] Почти чёрный фон с мягкой глубиной + абстрактный объект справа.
- [ ] Navbar с пунктами и chevron, справа action-ссылка.
- [ ] Mega menu: стеклянная большая панель (blur + border 1px + soft shadow).
- [ ] Внутри 3 колонки, секции с caps-заголовками.
- [ ] Пункты — строки: иконка в квадратике + title + серое описание.
- [ ] Hover эффекты очень мягкие, без ярких цветов.
- [ ] Бейджи маленькие, прозрачные, “пилюли”.

---

## 12) Что НЕ делать (чтобы не потерять стиль)

- Не использовать чистый чёрный (#000) и чистый белый (#fff) как основные.
- Не делать яркие кислотные акценты.
- Не делать толстые границы и резкие тени.
- Не перегружать интерфейс карточками без воздуха.
- Не смешивать разные стили иконок.
